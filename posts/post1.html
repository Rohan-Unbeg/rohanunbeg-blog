Mastering JavaScript Closures

By Rohan Unbeg | April 18, 2025 | 5 min read

What Are JavaScript Closures?

A closure is a function that "remembers" its lexical scope, even when executed outside that scope. This powerful feature enables data privacy, function factories, and more.

In simpler terms, closures allow a function to access variables from its outer scope, even after the outer function has finished executing.

Example of a Closure


function outerFunction() {
  let count = 0;
  return function innerFunction() {
    count++;
    console.log(count);
  };
}

const counter = outerFunction();
counter(); // 1
counter(); // 2

In this example, innerFunction forms a closure over count, retaining access to it even after outerFunction completes.

Practical Uses

  • Data Privacy: Create private variables that can't be accessed directly.
  • Function Factories: Generate functions with specific configurations.
  • Event Handlers: Maintain state in asynchronous operations.

Common Pitfalls

Closures can lead to memory leaks if not handled properly. For example, retaining large objects in a closure’s scope can prevent garbage collection.


function createHeavyClosure() {
  let largeArray = new Array(1000000).fill('data');
  return function() {
    console.log(largeArray[0]);
  };
}

Avoid this by nullifying references when the closure is no longer needed.

Real-World Example

Closures are often used in event listeners to maintain state:


function setupButton() {
  let clicks = 0;
  const button = document.querySelector('#myButton');
  button.addEventListener('click', () => {
    clicks++;
    console.log(`Button clicked ${clicks} times`);
  });
}

This keeps `clicks` private and persistent across clicks.

Closures are a cornerstone of advanced JavaScript programming. Mastering them unlocks powerful patterns for building robust applications.
